專案1：RPA架構診斷與技術自主化

2026年1月 ~ 2026年2月 | 1個月

核心挑戰

IT部門困境 — Power Automate RPA流程陷入維護地獄：黑盒流程難以維護，修改需要找原廠工程師，工作流執行不穩定經常超時。在無原廠文件情況下需快速自主化。

AI Agent協作應用

用Claude分析既有RPA流程邏輯，設計替代方案架構。Copilot生成Chrome Extension + Python FastAPI核心代碼。Gemini協助優化流程設計與文檔化。結果：1個月內完成技術自主化POC，比傳統外包方式快10倍。

成果指標

完成時間：1個月內完成技術自主化POC
架構改進：黑盒RPA → 高彈性、低延遲代碼架構
工具綁定風險：大幅降低，增加系統靈活性
技術債清償：用代碼化流程替代模擬UI自動化

技術方案

拆解既有RPA流程 → Chrome Extension (前端自動化) + Python FastAPI (後端處理) → 完全自主化。優勢：成本低無第三方許可證、可維護、可擴展、學習曲線平緩。

專案2-Gomaji：AI驅動營運流程自動化與報表生成系統

2026年3月 ~ 現在 | 進行中

業務問題

  • 症狀：PM與營運高度依賴人工處理日常任務
  • 痛點：報表與需求處理分散、低效率、頻繁重複
  • 成本：PM每日4小時用於重複性工作

解決方案

  • 將任務轉為可自動執行流程
  • AI輔助生成腳本與分析邏輯
  • Metabase標準化報表輸出
  • Multi-Agent 架構：Claude + Codex 分析與開發 Python 自動化腳本

AI成果指標

報表產出時間：4小時 → 30分鐘 (提升 87.5%)
PM日常任務：4小時 → 10分鐘 (自動化 97.5%)
決策支援速度：即時提供，可視化決策
涵蓋部門：營運、PM、市場、財務、行政 (5個部門)

核心價值

將「人工作業」轉為「系統資產」 — 建立可複用的營運流程。AI角色不是替代人類，而是將人類從重複工作解放出來做決策。

POC：NotebookLLM 組織級 Wiki 知識管理系統

2026年3月 ~ 進行中 | 概念驗證階段

問題背景

  • 現狀：知識分散在 Redmine、Google Docs、Slack 等多個平台
  • 痛點：員工查找資訊耗時，知識難以沉澱與複用
  • 機會：利用 AI 統一組織知識，建立智能查詢系統

解決方案

  • 核心技術：NotebookLLM 作為 AI 引擎
  • 知識源：Redmine 技術文檔 + Google Docs 內部維基
  • 集成方式：Google Docs API 自動同步 → NotebookLLM 知識庫 → AI 生成摘要
  • 交互方式：自然語言提問 → 即時知識檢索與合成

預期成果

  • ✓ 知識查詢時間：從 10-15 分鐘 → 30 秒內
  • ✓ 知識複用率提升：減少重複問題的回答
  • ✓ Onboarding 加速：新員工快速自助查詢
  • ✓ 組織智能資產：可複用的知識庫

當前進度

專案3：AI客服系統Prototype

2025年6月 ~ 2025年8月 | 2個月

AI應用創新

生成式AI浪潮下的實際應用 — 用LLM模型取代部分人工客服。Dify用於低代碼LLM工作流編排、知識庫檢索、多輪對話。Coze快速構建智能Bot跨渠道部署。n8n工作流串接客服回應 → 訂單系統 → CRM同步。

AI成果指標

客服自動回覆率：達到 70% (常見問題自動解答)
人工客服負擔：減少 60% (專注複雜問題)
客戶滿意度：實現 24小時 無人工延遲回應
成本節省：月度客服人力成本減少 30-40%

AI技術洞見

LLM不是萬能的：只適合高重複性低專業性的客服問題。關鍵是知識庫設計：AI品質取決於Prompt + 知識庫 + 上下文管理。混合模式最優：AI自動解答 + 人工複審 + 持續迭代。

專案3：銷售預測與會計自動化

2025年4月 ~ 2025年7月 | 3個月

AI/ML應用場景

結合預測模型與AI OCR解決財務部門的數據密集型工作：行銷部門每月手工預測銷售耗時且誤差大；會計每月需輸入百份傳票易出錯工作量大。

AI成果指標

銷售預測精度：月度預測誤差率降低 15% → 更準確的資源配置
會計傳票自動化：自動化率達 50%+
人工輸入錯誤：顯著減少，提升財務準確性
月度節省工時：80小時 (會計+行銷部門)

技術方案

預測層：ML時序預測模型基於歷史銷售資料、季節性因子。自動化層：AI OCR識別傳票 → 自動提取數字/簽章/日期 → 會計系統匯入。

專案4：全通路訂單管理系統

2021年10月 ~ 2023年6月 | 18個月

業務挑戰

公司使用不符需求的外部ERP，處理SHOPLINE、Shopee、MOMO三平台訂單時：人工校對、匹配物流成本高；旺季單量600+/日導致系統癱瘓；跨部門資訊斷層，成本難以控制。

成果指標

人工處理時間：從半天 → 1-1.5小時/日(90%+ 降低)
訂單錯誤率：下降 90%+
日均處理能力：從 200+ 提升至 600+(旺季3倍產能)
系統價值：每年節省 100萬+ 人工成本 + 倉庫協調費用

技術方案

後端：Xano NoCode平台 — 快速建模與業務邏輯編排。前端：Retool — 儀表板、訂單檢視、異常管理。整合層：API串接SHOPLINE/Shopee/MOMO/黑貓。工作流：自動匹配物流、生成配送單、庫存同步。

專案6：SHOPLINE × LINE CRM商品化

2024年7月 ~ 2025年3月 | 8個月

業務挑戰

與CRM廠商解約後需自主開發並探索SaaS化：行銷團隊無CRM工具客戶互動依賴試算表；每次行銷活動設定需2-3小時手動操作；無法跨平台統一管理客戶數據。

產品化：上架至SHOPLINE擴充功能商店，開創新收入來源

技術方案

功能：客戶管理、訊息推播、圖文選單、自動化行銷活動。技術棧：Xano後端 + Retool前端管理介面 + SHOPLINE API + LINE Bot。商業化：按訂閱計價 → 3家B2B客戶年合約收入。
